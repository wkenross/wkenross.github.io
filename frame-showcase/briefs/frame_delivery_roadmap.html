FRAME Delivery Roadmap

Updated: 2026-06-16

Overview

This roadmap shows how the FRAME automation framework is being managed as a technical project-management program. It complements the project-plan overview with delivery phases, release intent, Jira-backed evidence, and completion artifacts.

The roadmap is public-safe: it avoids private local paths, credentials, browser-session details, private job-application material, and account-specific configuration.

Roadmap

Phase Stage Target Status Evidence
Phase 0 Vision Define the FRAME technical project-management operating model. Done PlanMe, TrainMe, stage-gate model
Phase 1 V0 Scope Create registers, templates, and weekly status reporting. Done project, milestone, risk, dependency, and decision artifacts
Phase 2 P1 Build Seed Jira and publish the public-safe project-plan page. Done Jira issue range SCRUM-30 through SCRUM-35; project-plan URL
Phase 3 P1 Build Add roadmap, release, Jira status, RAID, milestone, and completion evidence artifacts. In Progress delivery artifact reports
Phase 4 P2 Handoff Generate delivery artifacts through PowerShell orchestration and Python operation scripts. In Progress repeatable delivery-artifact generator
Phase 5 Run Maintain weekly project status and public-safe updates. Next recurring reports and reviewed publish inputs

Jira-Backed Evidence

FRAME uses Jira as the visible Agile board while keeping durable reasoning in Markdown, CSV, and JSON artifacts.

Issue Range Meaning
SCRUM-5 through SCRUM-23 Initial Agile project-management operating system
SCRUM-24 through SCRUM-29 Search and product-owner toolkit implementation
SCRUM-30 through SCRUM-35 Technical project-management operating model

Jira provides the Agile tracking layer. FRAME-generated reports provide the client-facing timeline, release, RAID, milestone, and completion evidence.

Delivery Artifact Layer

The roadmap is part of a broader delivery evidence set:

Artifact Purpose
Delivery roadmap Show phases, delivery sequence, status, and evidence.
Release plan Show what is included and excluded in each release slice.
Jira status snapshot Show visible Agile issue ranges and sprint candidates.
RAID log Show risks, assumptions, issues, and dependencies.
Milestone tracker Show planned and completed delivery gates.
Completion evidence Tie finished work to files, reports, URLs, and Jira keys.

How The Roadmap Is Maintained

The working roadmap is generated inside the technical project-management workflow. The public page is staged through the publishing workflow after review.


FRAME registers and Jira reports
  -> PowerShell orchestration
  -> Python delivery-artifact generator
  -> reviewed Markdown
  -> wb_html
  -> GitHub Pages
