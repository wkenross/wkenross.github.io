Mobile Handoff Integration

Purpose

FRAME uses mobile devices as capture and review surfaces. The iPhone can help collect requirements, dictate notes, review prepared context, and hand off Markdown candidates. Windows remains the execution surface for PowerShell, Python, VS Code, browser automation, Office automation, SQL, and publication.

Handoff Model

The mobile loop is intentionally simple:

  1. Capture requirements, decisions, or field notes on the phone.
  2. Save the note into a mobile-safe inbox or context file.
  3. Review and normalize the note on Windows.
  4. Merge the requirement into the appropriate workspace documentation.
  5. Promote only curated public-safe material into publication inputs.

This makes mobile useful without treating it as a hidden automation engine.

What Belongs On Mobile

Good mobile inputs include:

  • short requirement notes
  • interview or customer follow-up points
  • ideas for PlanMe, CodeMe, RunMe, or PublishMe files
  • links to public reference material
  • quick review comments on already-sanitized Markdown
  • small context packets for a future Codex or ChatGPT session

Private credentials, tokens, raw mailbox exports, browser profile material, and customer data do not belong in public mobile handoff files.

iOS Boundary

The phone has a clear manual boundary. Windows scripts can prepare folders, generate context files, and write instructions, but iOS sign-ins, application installation, privacy prompts, file-provider permissions, and Shortcuts approvals remain manual.

That boundary is a feature. It keeps the operator in control of mobile identity and prevents the framework from implying silent control over phone apps.

Framework Integration

Mobile handoff content typically lands in one of three places:

Destination Use
install docs machine setup and mobile readiness instructions
workflow docs requirements for a specific workspace
publication intake public-safe Markdown candidates for the website

After review, the material can be routed to wb_html and then to the GitHub Pages showcase if it is useful for public readers.

Review Rules

  • keep public mobile examples generic
  • avoid raw phone screenshots unless they are inspected first
  • do not publish account identifiers or private iCloud/OneDrive paths
  • separate requirement capture from execution scripts
  • state clearly when an action requires manual iOS approval
