Road-Ready Outlook Sync Pattern

Purpose

Some FRAME workflows depend on Outlook Classic because it exposes mature desktop automation surfaces and synchronized mailbox folders. The road-ready sync pattern prepares mailbox folders and cleanup rules before travel or limited-connectivity work.

The goal is to make the synced mailbox easier to use on a laptop or phone without publishing private messages or relying on brittle one-off manual cleanup.

Operating Boundary

Outlook Classic is the automation host. Mobile Outlook and newer Outlook clients are treated as synchronized consumers of the resulting folder state.

That means:

  • folder creation and message movement happen through Outlook Classic
  • report-only runs come first
  • apply mode is explicit
  • deletes should be avoided or replaced with archive/move behavior
  • mobile clients are used to verify the synchronized result

Workflow Pattern

A safe road-ready cleanup flow:

  1. Inspect mailbox folders and candidate cleanup rules.
  2. Write a report without moving mail.
  3. Review the report for overly broad matches.
  4. Create or verify destination folders.
  5. Apply only approved moves.
  6. Confirm the folder state syncs to mobile.
  7. Keep a private report for audit and rollback decisions.

What To Publish

Public material can describe:

  • the report-first approach
  • the folder synchronization model
  • the difference between Outlook Classic execution and mobile review
  • why travel readiness matters for automation work
  • generic examples of category or sender-based routing

Public material should not include:

  • mailbox addresses
  • message subjects
  • sender lists
  • unsubscribe links
  • exact folder counts
  • screenshots with private account state
  • raw cleanup reports

Why It Matters

Road-ready sync supports the broader FRAME model. Requirements, job-search signals, customer notes, and workflow follow-ups often arrive through email. Having a clean synchronized mailbox makes it easier to capture those inputs and route them into the correct workspace when the operator returns to VS Code.

Acceptance Checks

  • report-only mode runs cleanly
  • apply mode is explicit
  • no hard-delete action is used for public examples
  • destination folders are described generically
  • private mailbox data stays out of the GitHub Pages pipeline
