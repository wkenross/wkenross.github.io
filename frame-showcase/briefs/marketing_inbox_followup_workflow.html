Marketing Inbox Follow-Up Workflow

Marketing automation is only useful if follow-up work remains visible after the first discovery pass.

The FRAME inbox pattern reads reviewed message metadata, classifies business signals, and proposes folder moves or next-step labels. It is designed for Outlook Classic workflows where real opportunities, rejections, replies, and status updates arrive beside ordinary inbox noise.

Folder Model

A public-safe folder model might include:


Inbox\Opportunities
Inbox\Opportunities\RFP
Inbox\Opportunities\Leads
Inbox\Opportunities\Follow_Up
Inbox\Opportunities\Rejected
Inbox\Opportunities\Next_Step

The exact names can change by client. The pattern stays the same:

  1. scan recent Inbox and Sent Items
  2. classify likely opportunity messages
  3. write a review report
  4. move or label only after approval

Classification Signals

Signal Likely action
RFP, request for proposal, bid, procurement route to RFP review
intro, referral, interested, can we talk route to lead follow-up
proposal sent, quote attached, estimate link to follow-up cadence
not selected, no longer needed, chose another vendor route to rejected
meeting, availability, next steps, approved route to next step

Why This Matters

Lead generation often fails after the initial list is built. The inbox workflow keeps the downstream process structured:

  • open opportunities are visible
  • rejections are not confused with active leads
  • next-step messages are easy to find
  • Sent Items can prove what was already proposed
  • follow-up dates can be fed back into the recommendation list

Safety Boundary

The public showcase should not include real message bodies, senders, mailbox names, unsubscribe links, or private opportunity details. Use synthetic examples and describe the review-first pattern.
