Public Energy And Economic Data Sources

Updated: 2026-06-09

Purpose

This page tracks public or licensable data sources that can replace private VisViva-style inputs in sanitized FRAME showcase workflows.

The target model is intentionally the same shape used by the Excel and SQL migration work:

  • metadata dimensions for products, contracts, indicators, regions, or markets
  • time-series observations by month, week, quarter, or year
  • cross-sectional point-in-time reporting
  • SQL-ready flat files
  • Power BI-ready exports after SQL validation

Recommended First Path

Use the EIA Open Data API as the first public energy-specific substitute. Pair it with FRED or BLS for economic context and with CFTC COT data when commodity positioning is useful. Treat CME DataMine as a later licensed-data option, not a first public dependency.

Source Catalog

Source Access Workflow Fit Status
U.S. Energy Information Administration Open Data API public API; API key may be required for production use Best first public replacement for private energy portal inputs because it provides energy-specific time series and metadata. recommended-first
FRED API public API with registered API key Good economic context layer for energy/commodity dashboards and Power BI pages. recommended
BLS Public Data API public API; no registration required for basic access Good price/inflation context for commodity and operating-cost reporting. recommended
CFTC Commitments of Traders Public Reporting Environment public reporting environment/API and downloadable formats Useful commodity-market context and public weekly data source for portfolio/reporting examples. recommended
World Bank Indicators API public API Useful for international context pages and certification/showcase examples. recommended-secondary
NOAA Climate Data Online API public API with token; documented request limits Useful weather-demand context for energy and commodity reporting demos. recommended-secondary
CME Group Market Data / DataMine licensed/commercial market data High-quality candidate for future paid/licensed examples; not the first public demo dependency. licensed-reference

Data Contract

Public data should land in the same SQL-friendly shape used by sr_sql_data:


customers.csv         -> synthetic customers/accounts or public region/economy dimension
contracts.csv         -> synthetic contracts or public product/indicator/market metadata
volumes.csv           -> time-series observations by contract/product/indicator/month
industry_updates.csv  -> source releases, topics, notes, and narrative summaries

Workbench Role

wb_html should publish the reviewed explanation and evidence. It should not be the first place private data is cleaned. The order is:

  1. clean and obfuscate local fr_ workflows
  2. promote reviewed sr_ workflow folders
  3. load synthetic or public data through sr_sql_data
  4. generate Power BI or CSV reporting outputs
  5. render reviewed Markdown into this HTML workbench

Review Notes

  • No API keys belong in this repository.
  • Licensed data should be documented but not embedded.
  • Public examples should cache only small synthetic or openly reusable samples.
  • Terms of use and attribution should be checked before public GitHub release.
