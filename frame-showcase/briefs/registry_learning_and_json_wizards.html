Registry Learning And JSON Wizards

FRAME uses JSON registries as practical memory for review-first automation.

Registries describe what the system knows:

  • source sites and search URLs
  • service offerings and capability tags
  • skills and evidence
  • wizard routing rules
  • known platform states
  • recommendation decisions
  • suppressions, skips, completions, and follow-up outcomes

Why JSON

JSON is simple enough for an operator to review and structured enough for scripts and AI-assisted scoring.

In practice:

  • CSV makes recommendation lists easy to inspect
  • JSON preserves state and routing rules
  • Markdown explains the decision trail
  • SQLite or DuckDB can be added later if history grows

Learning Loop

Each run improves the next run when decisions are written back:


collect signals
  -> score recommendations
  -> wizard prepares next action
  -> operator applies, skips, parks, or completes
  -> decision registry updates
  -> next recommendation list suppresses, boosts, or reroutes

Example Registry Shape


{
  "source": "public_rfp_directory",
  "industry": "operations_automation",
  "capability_tags": ["data_cleanup", "reporting", "workflow_automation"],
  "wizard": "rfp_review",
  "decision_history": {
    "status": "park_for_review",
    "reason": "Scope is strong but deadline and required attachments need review"
  }
}

Higher-Order AI Without Overpromising

The first improvement is not a custom model. It is disciplined data capture:

  • clear labels
  • consistent decisions
  • wizard outcomes
  • risk flags
  • human review notes

Once those examples exist, AI can help score future opportunities and explain why a row should be applied, skipped, parked, or routed to a specific wizard.
