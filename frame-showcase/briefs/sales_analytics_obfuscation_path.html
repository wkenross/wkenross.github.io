Sales Analytics Obfuscation Path

The sales analytics workflow can become a stronger GitHub showcase after a controlled obfuscation and synthetic-data pass.

What Can Publish First

  • functional descriptions
  • workflow maps
  • data-model explanations
  • orchestration diagrams
  • SQL/Power BI handoff requirements
  • sanitized screenshots or diagrams created from synthetic examples

What Must Stay Private Until Reviewed

  • Excel workbooks with real business values
  • portal scrape calls and private URLs
  • customer, carrier, contract, or employer labels
  • email notification examples containing real addresses or messages
  • browser profile, cookie, credential, and account-state details
  • scripts that still depend on private source systems

Obfuscation Steps

  1. Rename private workbook concepts to generic revenue-performance and

    2. projection names.

  2. Replace customer/employer-specific labels with FRAME Sales Automation terms.
  3. Scale or synthesize numeric values while preserving data shape.
  4. Replace private scrape sources with public proxy sources.
  5. Add SQL-ready normalized outputs keyed by contract and customer identifiers.
  6. Generate public-safe HTML pages through wb_html.
  7. Publish reviewed documentation through wb_github.

Future Public Code Gate

Code should move to GitHub only after it can run against synthetic or public inputs and no longer requires private portals, private workbooks, or hard-coded local paths.
