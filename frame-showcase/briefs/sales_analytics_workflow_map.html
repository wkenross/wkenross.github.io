Sales Analytics Workflow Map

The FRAME Sales Analytics workflow is organized around a functional map rather than the original private folder names.

Core Workflow

Sales analytics operations update point-in-time revenue performance, validate intraday contract adjustments, and coordinate multiple workbook update passes through common runtime settings.

The workflow pattern is:

  1. Load runtime configuration.
  2. Open or validate source workbooks.
  3. Refresh current-period performance.
  4. Validate contract/customer changes.
  5. Rebuild workbook reporting surfaces.
  6. Prepare normalized outputs for SQL and BI reporting.

Forecast Workflow

Fr_forecast reconciles historical performance and revenue growth against sales forecasts and projections. It preserves workbook inspection and pivot-validation utilities while moving toward synthetic public examples.

Scheduler Workflow

fr_scheduler describes schedule/calendar monitoring, automated notification, CRM update wizards, and legacy operations-management integration.

Shredder Workflow

Fr_shredder describes tools for shredding PDFs, scraping or exporting web source data, and consolidating extracted content into central reporting workbooks.

Month Shift Workflow

Month_Shift describes the controlled movement of the active reporting period, including headers, formulas, completed blocks, and reset blocks. Public examples should use alias/config guards before mutating workbook semantics.
