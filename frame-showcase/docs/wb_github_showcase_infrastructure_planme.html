WB GitHub Showcase Infrastructure PlanMe

Updated: 2026-06-09

Purpose

Define the web and GitHub infrastructure lane for publishing reviewed FRAME evidence without exposing private workbook data, private scrape targets, account state, or unsupported certification claims.

wb_html is the workbench that converts reviewed Markdown into simple static HTML. It sits downstream of the fr_ to sr_ obfuscation process and upstream of any GitHub Pages or cloud-hosted publication.

Role In The FRAME System

Layer Owner Role
fr_update local FRAME/update workspace Clean and obfuscate private Excel/scrape workflows.
sr_update OneDrive Search-side workspace Coordinate reviewed sr_ workflows for job-search evidence and showcase use.
sr_sql_data SQL workflow Load synthetic/public data, compare SQL aggregates to Excel pivots, and prepare Power BI outputs.
sr_for_hire client-facing workflow Package reviewed examples into proposals and GitHub showcase positioning.
wb_html web workbench Render reviewed Markdown/docs into a browseable static site.
GitHub Pages/cloud publication target Host only after privacy and claims review.

First Site Shape

Use a small static site before adopting a heavier framework:


wb_html/
  config/
    site_manifest.json
    source_catalog.json
    public_data_sources.json
  content/pages/
  docs/
  data/catalog/
  scripts/py/
  html/

Generated html/ output is reviewable locally and can later be copied into a repository /docs folder or a gh-pages branch.

GitHub Publication Path

  1. Build locally with scripts/py/build_site.py.
  2. Open html/index.html and review every page.
  3. Run the safety checklist in content/pages/safety_review.md.
  4. Confirm every linked source is marked public or limited-review.
  5. Copy generated HTML into the target GitHub Pages folder only after review.
  6. Publish privately first.
  7. Promote selected pages publicly after privacy, source, and claims review.

Public Data Replacement Track

The private VisViva-style scrape lane should be replaced with public or licensed data sources that fit the same model:

  • metadata dimensions
  • time-series facts
  • point-in-time cross-sectional reporting
  • SQL-ready flat files
  • Power BI-ready outputs

Initial candidates:

  • EIA Open Data API for energy fundamentals
  • FRED for economic context
  • BLS for price/labor/economic series
  • CFTC COT for commodity positioning
  • World Bank Indicators for global context
  • NOAA Climate Data Online for weather and energy-demand context
  • CME DataMine only as a later licensed-data candidate

What Counts As Publishable

Publishable:

  • sanitized Markdown
  • synthetic CSV samples
  • screenshots rebuilt from synthetic data
  • SQL schema and query examples
  • Power BI screenshots or exports from synthetic/public data
  • docs that truthfully describe skills, plans, and current status

Not publishable:

  • raw private workbooks
  • old portal scrape output
  • private URLs, account IDs, contacts, or emails
  • .pbix files containing private model data
  • Office/browser profile state
  • unearned certification claims
  • live API keys or tokens

Done Criteria

  • public_data_sources.json exists and documents candidate sources.
  • generated source catalog CSV exists under data/catalog.
  • public data page builds into the local HTML site.
  • sr_update.code-workspace includes wb_html.
  • publication remains review-first and private by default.
  • GitHub showcase docs point to wb_html as the web rendering lane.
