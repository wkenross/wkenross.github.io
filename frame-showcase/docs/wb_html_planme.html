WB HTML Site PlanMe

Updated: 2026-06-09

Purpose

Create a lightweight web workflow for publishing selected FRAME documentation as a polished, review-first static site.

The site should make the growing collection of PlanMe, TrainMe, CertifyMe, Agile, SQL, Power BI, CRM, and safety documents easier to browse without losing the Markdown-first operating model.

Goals

  • convert curated Markdown into GitHub-hostable HTML
  • keep source Markdown as the working surface
  • give PlanMe, TrainMe, CertifyMe, resume, and tool pages distinct but related

    • looks

  • support SQL and Power BI showcase pages using synthetic data
  • support Agile/Jira certification evidence pages
  • support CRM/HubSpot/Salesforce planning pages without private tenant details
  • keep publication review-first and limited-audience until scrubbed

Non-Goals

  • publish private workbook inputs
  • publish live Jira, Salesforce, HubSpot, Outlook, or SharePoint URLs
  • publish .pbix files before embedded data and connection strings are

    • inspected

  • replace the owning workflow folders
  • create a decorative site before the content is useful

Source Workflow


FRAME Markdown docs
        |
source catalog and review tags
        |
curated public-safe Markdown in wb_html/content
        |
build_site.py
        |
wb_html/html
        |
GitHub Pages or private review link

Folder Shape


wb_html/
  assets/
    css/
    img/
  config/
  content/
    pages/
  docs/
  html/
  md/
  pdf/
  scripts/
    py/

content/pages holds curated site copy. md and pdf are reserved for future exported artifacts. html is generated output.

Sub-Workflows

1. Site Shell

  • build navigation
  • build home page
  • build topic pages
  • copy shared CSS and image assets

Done when html/index.html opens locally and links to all staged pages.

2. Markdown Curation

  • identify source docs
  • decide whether each page is public, limited, internal, or blocked
  • copy only public-safe ideas into content/pages
  • keep source paths in config/source_catalog.json

Done when every page has a source and review status.

3. Style System

  • PlanMe pages look strategic and roadmap-oriented
  • TrainMe pages look instructional and step-based
  • CertifyMe pages look evidence/checklist-oriented
  • resume pages stay compact and hiring-manager friendly
  • Jira, Salesforce, HubSpot, and Power BI pages share the same neutral FRAME

    • style rather than copying each vendor UI

Done when CSS supports these page kinds without separate one-off templates.

4. SQL And Power BI Showcase

  • connect SQL PlanMe/TrainMe/CertifyMe to Power BI PlanMe/TrainMe/CertifyMe
  • show how SQL exports improve Excel BI reporting
  • include only synthetic screenshots or screenshots produced from synthetic

    • data

  • link PL-300 certification readiness to real report examples

Done when the site has one SQL/Power BI page with a clear path to sample reports.

5. GitHub Pages Publishing

  • build into html
  • review locally
  • copy or publish the output through the chosen GitHub Pages path
  • keep private source files out of the published branch/folder

Done when a reviewer can browse the generated site without needing local private data.

Acceptance Criteria

  • wb_html is included in sr_update.code-workspace.
  • The first build creates html/index.html.
  • The style guide defines PlanMe, TrainMe, CertifyMe, resume, and tool-page

    • treatments.

  • Source catalog tracks safety status.
  • SQL/Power BI certification path is represented.
  • External tool pages use generic language and scrubbed examples.
  • Publication remains review-first.
