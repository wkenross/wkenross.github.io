WB HTML Style Guide

Updated: 2026-06-09

Purpose

Give FRAME web pages a consistent style while allowing different document types to feel distinct.

The visual tone should be calm, credible, and evidence-oriented. It should feel like an operations and data-product workbench, not a glossy marketing page.

Page Kinds

Kind Use Visual Treatment
planme strategy, roadmap, decisions blue accent, roadmap label, larger section headings
trainme practical learning, step-by-step work green accent, numbered routines, instructional tone
certifyme readiness, credentials, evidence amber accent, checklist and evidence table emphasis
resume role-specific career material compact typography, tight spacing, proof bullets
tool Jira, Salesforce, HubSpot, Power BI neutral accent, integration and safety notes
default general docs quiet neutral treatment

Shared Rules

  • Use one restrained FRAME style across Jira, Salesforce, HubSpot, and Power BI

    • pages.

  • Do not imitate vendor dashboards or account screens.
  • Use generic tenant/account wording unless a private reviewer explicitly needs

    • the real path.

  • Prefer tables, checklists, and short evidence sections.
  • Keep pages useful when printed or saved as PDF.
  • Avoid screenshots until they are scrubbed.

Layout

  • left navigation on desktop
  • top navigation on narrow screens
  • readable content width
  • no nested cards
  • no decorative gradient blobs
  • code and commands in clearly framed blocks
  • tables scroll on small screens

Document Conversion Notes

PlanMe pages should lead with purpose, goals, non-goals, phases, and acceptance criteria.

TrainMe pages should lead with why a package or approach was chosen, then show the standard loop, commands, practical lessons, and done criteria.

CertifyMe pages should lead with credential target, official references, evidence map, readiness checklist, and truthful resume language.
