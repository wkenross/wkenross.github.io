WB Public Data Source PlanMe

Updated: 2026-06-09

Purpose

Plan the replacement of private energy/commodity scrape inputs with public or licensable sources that can support the same FRAME workflow pattern:


public data -> Python loader -> Excel/CSV review -> SQL model -> Power BI/HTML evidence

Recommended First Candidate

Start with the U.S. Energy Information Administration Open Data API because it is energy-specific and naturally fits the migrated Excel/SQL reporting model.

Use FRED and BLS for economic context, CFTC COT for commodity positioning, NOAA Climate Data Online for weather-demand context, and World Bank indicators for broader global examples. CME market data is useful as a reference path but should not be a first public demo dependency because it is licensed/commercial.

Data Shape

The public-source workflow should feed the same sr_sql_data model:

FRAME file Meaning
customers.csv synthetic customer/account/region/economy dimension
contracts.csv synthetic contract/product/indicator metadata
volumes.csv monthly/weekly/quarterly time-series observations
industry_updates.csv source/release/topic/summary notes

The names can remain generic even when the source is public. The demonstration is the repeatable data model, not a claim that public data exactly replaces the old private portal.

Implementation Phases

Phase 1 - Catalog

  • maintain config/public_data_sources.json
  • generate data/catalog/public_data_sources.csv
  • generate content/pages/public_energy_data.md
  • render the page through build_site.py

Phase 2 - Small Public Sample

  • pick one EIA series or endpoint
  • cache a tiny sample as CSV
  • map it into volumes.csv style fields
  • keep API keys out of the repo

Phase 3 - SQL Load

  • load mapped public/synthetic samples through sr_sql_data
  • compare SQL outputs to simple Excel pivots
  • export CSVs for Power BI

Phase 4 - HTML Evidence

  • render the source explanation, schema, query outputs, and screenshots into

    • wb_html

  • publish only after review

Safety And Attribution

  • use official source links in docs
  • check terms of use before public release
  • avoid embedding licensed data
  • do not publish API keys
  • mark pages as limited-review until data rights and attribution are clear

Done Criteria

  • a public data source catalog is generated
  • at least one public source is mapped to the SQL-ready shape
  • sample data is small, synthetic, or clearly permitted for reuse
  • Power BI evidence uses public/synthetic data
  • HTML pages explain the source and model clearly
